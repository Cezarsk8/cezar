# Release Notes - v2.1.0 Internationalization (EN/PT/ZH)

**Release Date**: April 9, 2026
**User Story**: US004-Internationalization

---

## What's New

### Language Switcher
- EN | PT | ZH buttons fixed in the top-right corner
- Follows Midnight Architect design (dark background, cyan active state)
- Works on all screen sizes

### Auto-Detection
- First visit auto-detects browser language (pt-BR → PT, zh-CN → ZH, else → EN)
- Preference saved in localStorage and persists across visits

### Full Translation Coverage (83 keys per locale)
- Hero: subtitle, metric labels, CTA buttons
- Projects: types, taglines, feature descriptions, badges
- Skills: category names
- Experience: job titles, locations, all achievement bullets (RAZAC 4 areas, Kestraa, Sinerlog, Correios)
- Education: degree names, types
- Contact: form labels, placeholders, success/error messages
- Footer: navigation links, copyright
- Meta tags, OG tags, JSON-LD update per locale

### Chinese Support
- Noto Sans SC font loaded for proper CJK rendering
- `html[lang="zh"]` font-family override for body and headings

### Other Changes
- Removed "Available for opportunities" badge from hero
- Typing effect restarts with translated roles when switching language

## Technical Details

### New Files
- `i18n/en.json` — English translations (source of truth)
- `i18n/pt.json` — Portuguese translations
- `i18n/zh.json` — Chinese translations
- `js/i18n.js` — Translation engine (auto-detect, localStorage, DOM injection)

### Modified Files
- `index.html` — 83 `data-i18n` attributes added, language switcher component
- `js/main.js` — Typing effect and form messages read from i18n data
- `css/styles.css` — Language switcher styles, CJK font overrides, removed badge CSS
- `css/print.css` — Hide language switcher in print

### i18n Architecture
- `data-i18n="key"` → replaces textContent
- `data-i18n-html="key"` → replaces innerHTML (for content with HTML entities)
- `data-i18n-ph="key"` → replaces placeholder attribute
- `data-i18n-list="key"` → renders array items as `<li>` elements

## Breaking Changes
- None
